# Template Sync

[Back to README](../README.md)

After initial setup, keep projects in sync with template changes using two skills.

## `/sync-template` (downstream: template -> project)

Pull template updates into your project. Reads `.claude/template-manifest.json`, compares file hashes, and:

- **Auto-updates** unmodified files (template changed, project didn't customize)
- **Flags conflicts** for files you've customized (shows diff, you decide)
- **Detects new files** added to the template since last sync

## `/contribute-upstream` (upstream: project -> template)

Push project improvements back to the template. Finds locally modified template-sourced files, helps you:

- **Classify changes** as project-specific or generalizable
- **Reverse placeholder replacement** (concrete values back to `{{PLACEHOLDER}}`)
- **Propagate** to all template variants for shared files (AGENT_TEAM.md, settings.json)

## `/commit` Integration

When committing, if template-sourced files are modified, the commit workflow:

- Auto-updates the manifest (`locallyModified: true`)
- Prints a reminder: "Run `/contribute-upstream` when ready"

## Manifest: `.claude/template-manifest.json`

Generated automatically by `setup-project.ps1`. Tracks:

- Which **template variant** was applied
- **Path** to the claude-code-toolkit repo
- **Placeholder values** used (for reverse-mapping)
- **Per-file content hashes** and modification status

## Prerequisites

Both `/sync-template` and `/contribute-upstream` require `.claude/template-manifest.json` to be present. This file is generated by the setup script during initial template application. If you set up manually, these skills will not be available until the manifest is created.
