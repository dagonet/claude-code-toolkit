# Template Sync

[Back to README](../README.md)

After initial setup, keep projects in sync with template changes using two skills. Both skills delegate all deterministic operations (hashing, diffing, placeholder replacement, three-way merge) to the **template-sync-tools** MCP server — the AI handles only user interaction and judgment calls.

## Prerequisites

- `.claude/template-manifest.json` must exist (generated by the setup script)
- **template-sync-tools** MCP server must be registered (see [`mcp-servers/HOWTO.md`](../mcp-servers/HOWTO.md))
- If template-sync-tools is not available, the skills will not function

## `/sync-template` (downstream: template -> project)

Pull template updates into your project. The skill calls MCP tools to classify each file:

| Template changed? | Project changed? | Status | Action |
|---|---|---|---|
| No | No | `UP_TO_DATE` | Skip |
| No | Yes | `PROJECT_CUSTOM` | Skip (your customization preserved) |
| Yes | No | `AUTO_UPDATE` | Auto-apply template update |
| Yes | Yes | `CONFLICT` | Three-way merge, you resolve |

For conflicts, the MCP server reconstructs the common ancestor via `git show` and produces a three-way merge. If the merge is clean, it's offered for approval. If it has conflict markers, you resolve manually.

## `/contribute-upstream` (upstream: project -> template)

Push project improvements back to the template. The skill:

1. Identifies locally modified template-sourced files (skips `PROJECT_CONTEXT.md`)
2. Shows diffs for each — you classify as generalizable or project-specific
3. Reverses placeholder replacement deterministically (longest values first)
4. Checks cross-variant file identity and propagates to matching variants
5. Stages changes in the template repo (does not commit — you review first)

## `/commit` Integration

When committing, if `.claude/template-manifest.json` exists, the commit workflow calls `template_compute_status` to detect drift:

- **Locally modified files**: Prints a reminder to run `/contribute-upstream`
- **Template updates available**: Prints a reminder to run `/sync-template`

This is informational only — it never blocks a commit.

## Manifest v2: `.claude/template-manifest.json`

Generated by `setup-project.ps1` (Windows) or `setup-project.sh` (Linux/macOS). The v2 format tracks:

| Field | Purpose |
|-------|---------|
| `version` | Schema version (currently `2`) |
| `variant` | Which template was applied |
| `templateRepo` | Path to the claude-code-toolkit repo |
| `lastSynced` | Git commit hash of last sync point |
| `placeholders` | Concrete values used during setup (for reverse-mapping) |
| `files.*.templateHash` | SHA-256 of template file after placeholder replacement |
| `files.*.templateRawHash` | SHA-256 of template file before placeholder replacement |
| `files.*.localHash` | SHA-256 of project file at last sync (common ancestor for merge) |
| `files.*.locallyModified` | Whether the file has been modified since last sync |

The `templateRawHash` field detects template changes without recomputing placeholder replacement. The `localHash` field serves as the common ancestor for three-way merge during conflict resolution.

v1 manifests (missing `version`, `templateRawHash`, `localHash`) are auto-migrated to v2 by the `template_load_manifest` MCP tool.
